Inicío do OpenPost

Muito Obrigado

Antes de mais nada agradeço a curiosidade para acessar esse pequeno e escalável projeto. Inicialmente, vamos comentar sobre o que é o OpenPost.

OpenPost - Postagem de forma aberta

Bem, assim como o nome diz, o OpenPost é literalmente um website onde poderá ser feito postagens de forma aberta, o que isso significa é que não terá assinatura, poderá ser postado sobre variados tópicos, mesmo que tenha uma preferencia por tecnologia, e terá como principal objetivo espalhar opiniões, claro que haverá regras para um ambiente mais saudável

O Diferencial

Mas, para um projeto conseguir prosseguir ele precisa ter diferenciais, e estes são os diferenciais do OpenPost e as explicações de cada um deles.

OpenSource

Inicialmente, é interessante nos questionarmos, o que seria uma plataforma de posts opensource? Isso significa que todos podem ter acesso ao código fonte das postagens e utilizá-lo livremente, e com o acesso ao código fonte vem o segundo diferencial

A prática leva a perfeição

Um ponto crucial da plataforma é a maneira que novos posts são feitos, através de Pull Requests no GitHub do projeto! Isso significa que em todos os posts o usuário não apenas tem que saber programar em HTML, CSS e JS e que servirá como prática para eles! Um pequeno Q&A de perguntas que podem ter surgido:

Por que não utilizar frameworks como React?

A prática de framework não é ruim, mas todos em como base o trio web (HTML, CSS, JS) e por isso existe a necessidade de programar nelas. Além de ser a unidade básica para web as três podem ser extremamentes diversificadas utilizando de tags para utilizar, por exemplo, o Vue. Além disso existe o fato de que apenas com uma base forte será possível realmente dominar estas outras técnologias.

Mas e sobre as múltiplas técnologias e CSS em diferentes Posts?

Isso é um dos diferenciais do Open Post, a capacidade de ser totalmente diferente a cada Post realizado, na página inicial e no primeiro post foram utilizados brancos para expressar um nulo sobre estilos. O que isso quer dizer é que cada post pode ter um estilo único, fontes únicas, assuntos únicos, técnologias únicas, cada post é único e isso forma a comunidade aberta não só do OpenPost mas, também, do OpenSource.

E sobre aqueles que não sabem programar?

Bem, pelo objetivo do OpenPost ser não só para os programadores práticar mas também para compartilhar conhecimentos com o interesse em tentar postar novos conteúdos esperamos que crie uma maior curiosidade sobre ela e que, a partir dela, surjam novos programadores preparados para o mercado e adeptos a comunidade OpenSource.
